💳 Transaction-Based Payment Simulation
📌 Task Description

Simulate an online payment transaction system that:

Deducts balance from a user account

Adds the amount to a merchant account

Performs COMMIT on successful transaction

Performs ROLLBACK on failure

This task demonstrates basic transaction management concepts used in digital payment systems.

🛠 Technologies Used

HTML

CSS

JavaScript

⚙ Working Logic

User enters payment amount.

System checks if sufficient balance is available.

If balance is sufficient:

Deduct amount from user.

Add amount to merchant.

Display Transaction Successful (COMMIT).

If balance is insufficient:

No changes are made.

Display Transaction Failed (ROLLBACK).

🏦 Real-Time Usage

This concept is used in:

Online Banking

UPI Payments

ATM Withdrawals

E-commerce Payment Systems

🎯 Objective

To understand how transaction control mechanisms like COMMIT and ROLLBACK work in real-time financial applications.
